# Хеширование пароля — MaHash9

MaHash9 — простая хеш-функция, в которой используется таблица подстановки из криптоалгоритма Skipjack и циклический сдвиг влево на 14 бит.

Начальное значение хеша равно длине строки:
```swift
var hash = UInt32(bytes.count)
```

Это означает, что даже при одинаковом наборе символов строки разной длины дадут разный хеш.

На каждом шаге выполняется три операции: инверсия текущего хеша, нелинейное смешивание через сдвиги и вычитание табличного значения:
```swift
hash = lrot14(~hash &+ ((hash << 6) ^ (hash >> 11))) &- UInt32(sTable[tableIdx])
```

`~hash` — побитовая инверсия всех 32 бит. Это гарантирует, что даже одинаковые символы на разных позициях дадут разный промежуточный результат.

`(hash << 6) ^ (hash >> 11)` — перемешивание: биты сдвигаются влево и вправо, затем складываются по XOR. Это создаёт зависимость каждого выходного бита от множества входных.

`lrot14` — циклический сдвиг влево на 14 бит. В отличие от обычного сдвига, биты не теряются, а перемещаются на другую сторону числа:
```swift
private static func lrot14(_ x: UInt32) -> UInt32 {
    return (x << 14) | (x >> 18)
}
```

Индекс в таблице подстановки вычисляется с учётом позиции символа:
```swift
let tableIdx = Int((UInt32(byte) + UInt32(i)) & 0xFF)
```
Прибавление позиции `i` к значению байта гарантирует, что одинаковые символы в разных позициях строки дадут разные индексы и, следовательно, разный вклад в хеш.

Таблица подстановки `sTable` содержит 256 байт — это фиксированная перестановка значений 0–255, заимствованная из алгоритма Skipjack. Она вносит нелинейность: результат не зависит линейно от входа.

Итоговый хеш преобразуется в строку из 8 шестнадцатеричных символов:
```swift
static func hexString(_ password: String) -> String {
    return String(format: "%08X", hash(password))
}
```

Хеш используется как seed генератора псевдослучайных чисел:
```swift
let seed = UInt64(MaHash9.hash(password))
```

Одинаковый пароль всегда даёт одинаковый seed, а значит — одинаковую гамма-последовательность. Именно это позволяет отправителю и получателю с одним паролем зашифровать и расшифровать данные.

# Потоковый XOR-шифр

Потоковый шифр работает по принципу наложения гаммы: каждый байт исходных данных складывается по XOR с очередным байтом псевдослучайной последовательности (гаммой), которую вырабатывает генератор.

```
зашифрованный_байт = исходный_байт XOR гамма_байт
```

Ключевое свойство XOR: операция симметрична. Если применить её дважды с одним и тем же ключевым потоком, получим исходные данные:

```
(исходный XOR гамма) XOR гамма = исходный
```

Поэтому функции шифрования и дешифрования идентичны — обе вызывают один и тот же метод `process`:
```swift
static func process(data: [UInt8], password: String, generator: GeneratorChoice) -> [UInt8] {
    let seed = UInt64(MaHash9.hash(password))
    // ...
    for byte in data {
        output.append(byte ^ gen.nextByte())
    }
}
```

Генератор инициализируется один раз перед началом обработки и непрерывно вырабатывает гамму байт за байтом — поток остаётся непрерывным для всего файла любого размера.

# Квадратичный конгруэнтный генератор

Квадратичный конгруэнтный генератор вырабатывает псевдослучайную последовательность по формуле с квадратичной зависимостью от предыдущего состояния:

```
X_n = (a · X²_{n-1} + b · X_{n-1} + c) mod m
```

Наличие квадратичного члена делает последовательность менее предсказуемой по сравнению с линейным генератором: восстановить параметры только по выходным значениям значительно сложнее.

Параметры генератора:
```swift
private let a: Int64 = 1_664_525     // множитель квадратичного члена
private let b: Int64 = 22_695_477    // множитель линейного члена
private let c: Int64 = 1             // свободный член
private let m: Int64 = 2_147_483_647 // модуль — простое число 2^31 - 1
```

Модуль `m = 2³¹ − 1` является числом Мерсенна. Использование простого числа в качестве модуля обеспечивает максимально длинный период последовательности до её повторения.

Чтобы избежать переполнения типа `Int64` при возведении в квадрат, вычисление разбито на шаги с промежуточным взятием остатка:
```swift
let sq   = (state % m) * (state % m) % m    // X²_{n-1} mod m
let term = (a * sq % m + b * state % m + c) % m
state = term
```

Для получения одного байта генератор вызывается 8 раз, и каждый раз берётся младший бит нового состояния. Биты собираются в байт побитовым сдвигом:
```swift
func nextByte() -> UInt8 {
    var result: UInt8 = 0
    for i in 0..<8 {
        // ... вычисление нового состояния ...
        result |= UInt8(state & 1) << i
    }
    return result
}
```

# Генератор «Стоп-Пошел»

Генератор «Стоп-Пошел» — комбинирующий генератор на основе трёх сдвиговых регистров с линейной обратной связью (LFSR). Нелинейность достигается за счёт управляемого тактирования.

Структура генератора:
- **LFSR-1** — управляющий регистр, тактируется на каждом шаге.
- **LFSR-2** — рабочий регистр, тактируется только если предыдущий выход LFSR-1 был равен **1** («Пошел»).
- **LFSR-3** — рабочий регистр, тактируется только если предыдущий выход LFSR-1 был равен **0** («Стоп»).
- Выходной бит — XOR выходов LFSR-2 и LFSR-3.

Каждый LFSR реализован в конфигурации Галуа: если младший бит равен 1, регистр сдвигается вправо и к результату применяется XOR с маской обратной связи; иначе — просто сдвиг:
```swift
private func clockLFSR(_ reg: UInt32, mask: UInt32) -> UInt32 {
    return (reg & 1 == 1) ? (reg >> 1) ^ mask : reg >> 1
}
```

Маски выбраны из таблиц примитивных многочленов, что гарантирует максимальный период каждого регистра:
```swift
private let mask1: UInt32 = 0x8000_0057  // LFSR-1: отводы (32,7,5,3,2,1,0)
private let mask2: UInt32 = 0x4000_0004  // LFSR-2: 31-битный
private let mask3: UInt32 = 0x1000_0002  // LFSR-3: 29-битный
```

Ключевой момент: решение о тактировании принимается на основе выхода LFSR-1 до его собственного тактирования на текущем шаге — сохранённое значение `out1`:
```swift
func nextByte() -> UInt8 {
    var result: UInt8 = 0
    for i in 0..<8 {
        let out1 = reg1 & 1                      // сохраняем до тактирования
        reg1 = clockLFSR(reg1, mask: mask1)      // LFSR-1 тактируется всегда
        if out1 == 1 {
            reg2 = clockLFSR(reg2, mask: mask2)  // «Пошел»
        } else {
            reg3 = clockLFSR(reg3, mask: mask3)  // «Стоп»
        }
        let bit = UInt8((reg2 & 1) ^ (reg3 & 1))
        result |= bit << i
    }
    return result
}
```

Регистры имеют разную длину (32, 31 и 29 бит) — их периоды взаимно просты, поэтому итоговый период генератора близок к произведению трёх индивидуальных периодов, что даёт очень длинную последовательность до повторения.

Нулевое состояние любого регистра недопустимо: заполненный нулями LFSR будет бесконечно выдавать нули. Поэтому при инициализации нулевые значения заменяются фиксированными константами:
```swift
if reg1 == 0 { reg1 = 0xACE1_CAFE }
if reg2 == 0 { reg2 = 0xDEAD_BEEF }
if reg3 == 0 { reg3 = 0xFEED_F00D }
```

# Конгруэнтные генераторы — Квадратичный конгруэнтный генератор

Квадратичный конгруэнтный генератор — это усложнённая версия линейного конгруэнтного генератора. В отличие от линейного, в формуле присутствует квадратичный член, что делает последовательность более сложной и менее предсказуемой.

Принцип работы:

Каждое следующее число последовательности вычисляется по предыдущему, но с использованием квадрата. Выходным битом является младший бит полученного значения.

Xₙ = (a × Xₙ₋₁² + b × Xₙ₋₁ + c) mod m

Здесь:
- **a** — множитель квадратичного члена
- **b** — множитель линейного члена  
- **c** — свободный член (инкремент)
- **m** — модуль (как правило простое число для лучшего периода)
- **X₀** — начальное значение (seed)

## 1. Инициализация
```swift
init(seed: Int64) {
    self.state = seed % m
    if self.state <= 0 { self.state = 1 }
}
```
Начальное значение приводится к допустимому диапазону `[1, m-1]`. Ноль недопустим, так как при нулевом состоянии вся последовательность вырождается в нули из-за свойства умножения.

## 2. Основные вычисления
```swift
let sq   = (state % m) * (state % m) % m    // X²_{n-1} mod m
let term = (a * sq % m + b * state % m + c) % m
state = term
```
Вычисление разбито на шаги с промежуточным взятием остатка, чтобы избежать переполнения типа `Int64`. Сначала вычисляется квадрат текущего состояния по модулю m, затем прибавляются линейный и свободный члены.

## 3. Возврат бита
```swift
return UInt8(state & 1)
```
Выходным псевдослучайным битом служит младший бит нового состояния. Операция `& 1` — это маска, выделяющая только последний бит числа в двоичном представлении.

## Параметры генератора
```swift
private let a: Int64 = 1_664_525     // множитель квадратичного члена
private let b: Int64 = 22_695_477    // множитель линейного члена
private let c: Int64 = 1             // свободный член
private let m: Int64 = 2_147_483_647 // модуль (простое число 2^31 - 1)
```
Модуль `m = 2³¹ − 1` является числом Мерсенна и обеспечивает максимальный период генератора. Коэффициенты `a` и `b` подобраны так, чтобы последовательность проходила через как можно большее количество различных состояний перед повторением.

По сравнению с линейным конгруэнтным генератором квадратичный обладает большей стойкостью к взлому, потому что нелинейная зависимость значительно усложняет задачу восстановления параметров генератора по выходной последовательности.

# Генератор «Стоп-Пошел» (Stop-and-Go Generator)

Генератор «Стоп-Пошел» — это комбинирующий генератор на основе трёх сдвиговых регистров с линейной обратной связью (LFSR). Нелинейность достигается за счёт того, что один регистр управляет тактированием двух других.

Принцип работы:

- **LFSR-1** — управляющий регистр, тактируется на каждом шаге.
- **LFSR-2** — рабочий регистр, тактируется только если предыдущий выход LFSR-1 был равен **1** («Пошел»).
- **LFSR-3** — рабочий регистр, тактируется только если предыдущий выход LFSR-1 был равен **0** («Стоп»).
- Выходной бит = **XOR(выход LFSR-2, выход LFSR-3)**.

## 1. Сдвиговый регистр с линейной обратной связью (LFSR)

LFSR — это регистр из n битов, у которого новый крайний бит вычисляется как XOR нескольких выбранных битов регистра (отводная последовательность). Реализуется в конфигурации Галуа: если младший бит равен 1, регистр сдвигается вправо и применяется XOR с маской обратной связи; иначе — просто сдвиг.

```swift
private func clockLFSR(_ reg: UInt32, mask: UInt32) -> UInt32 {
    if reg & 1 == 1 {
        return (reg >> 1) ^ mask  // сдвиг + применение обратной связи
    } else {
        return reg >> 1            // просто сдвиг
    }
}
```

## 2. Отводные последовательности и маски

Для достижения максимального периода многочлен, задаваемый отводной последовательностью, должен быть примитивным по модулю 2. Маски выбраны из таблиц примитивных многочленов:

```swift
private let mask1: UInt32 = 0x8000_0057  // LFSR-1: отводы (32,7,5,3,2,1,0)
private let mask2: UInt32 = 0x4000_0004  // LFSR-2: 31-битный регистр
private let mask3: UInt32 = 0x1000_0002  // LFSR-3: 29-битный регистр
```

Регистры разной длины гарантируют, что их периоды взаимно просты, а итоговый период генератора равен произведению периодов всех трёх LFSR.

## 3. Инициализация

```swift
init(seed: UInt64) {
    reg1 = UInt32(seed & 0xFFFF_FFFF)
    reg2 = UInt32((seed >> 8)  & 0xFFFF_FFFF)
    reg3 = UInt32((seed >> 16) & 0xFFFF_FFFF)

    if reg1 == 0 { reg1 = 0xACE1_CAFE }
    if reg2 == 0 { reg2 = 0xDEAD_BEEF }
    if reg3 == 0 { reg3 = 0xFEED_F00D }
}
```

Из одного числа seed разными битовыми сдвигами получаем три разных начальных состояния. Нулевое состояние недопустимо — регистр, заполненный нулями, будет бесконечно выдавать нули, поэтому заменяем его фиксированными ненулевыми значениями.

## 4. Генерация бита

```swift
func nextBit() -> UInt8 {
    let out1 = reg1 & 1             // сохраняем текущий выход LFSR-1

    reg1 = clockLFSR(reg1, mask: mask1)  // LFSR-1 тактируется всегда

    if out1 == 1 {
        reg2 = clockLFSR(reg2, mask: mask2)  // «Пошел» — LFSR-2 тактируется
    } else {
        reg3 = clockLFSR(reg3, mask: mask3)  // «Стоп» — LFSR-3 тактируется
    }

    return UInt8((reg2 & 1) ^ (reg3 & 1))   // XOR выходов двух рабочих регистров
}
```

Ключевой момент: решение о тактировании принимается на основе **предыдущего** выхода LFSR-1, то есть значения `out1`, сохранённого до шага тактирования. Это обеспечивает нелинейную зависимость выхода от состояния регистров.

## Преимущества и ограничения

Генератор «Стоп-Пошел» обладает нелинейностью за счёт управляемого тактирования, что делает его значительно более стойким к криптоанализу по сравнению с одиночным LFSR. Период генератора при взаимно простых длинах регистров близок к произведению их индивидуальных периодов. Тем не менее, как и другие LFSR-based генераторы, он уязвим при известных начальных условиях и не рекомендуется как единственное средство защиты в современных криптосистемах.
